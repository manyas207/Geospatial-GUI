## Follow-up chat

### `POST /api/followup`

Ask Ollama about the current dashboard or city context.

**Content-Type:** `application/json`

```json
{
  "question": "Which city is hottest?",
  "context": {
    "model": "equity",
    "summary": "11-city urban heat and equity demo.",
    "stats": { "active_city": "Phoenix, AZ" },
    "logs": "",
    "raster": "",
    "tract_layer_token": "Z3Bn...",
    "demo_cities": [{ "name": "Phoenix, AZ", "peak_lst_C": 42.3 }],
    "demo_overview": { "hottest_city": "Phoenix, AZ", "peak_lst_C": 42.3 }
  }
}
```

**Response:**

```json
{ "answer": "Phoenix, AZ has the highest demo LST (42.3Â°C)." }
```

**Rate limits:** Per client IP (default 15 requests per 60 seconds). Configure with `CHAT_RATE_LIMIT_MAX`, `CHAT_RATE_LIMIT_WINDOW`, and `CHAT_MAX_QUESTION_LENGTH` in `.env`. Returns `429` with `Retry-After` when exceeded.

**Project context:** When `context.project_id` is set, the server attaches `city_comparison` from the project manifest. For tract queries, use `tract_layer_token` as either a city-layers cache token or `{project_id}:{city_key}` for enriched project GPKGs.

**Demo context:** When `context.demo_cities` is set (no `project_id`), cross-city comparison uses placeholder LST values.

**Context fields:** `raster` must be a string (use `""` when no GeoTIFF path is available). Optional fields may be omitted or set to `null` except where noted in `DashboardContext`.

**Errors:** `400` (empty or overlong question), `422` (invalid request body â€” e.g. wrong type for `context.raster`), `429` (rate limit), `500` (server error).

## Multi-city LST projects

### `POST /api/projects`

Create an empty multi-city LST project.

```json
{ "name": "My LST City Project" }
```

**Response:** manifest with `id`, `cities`, `ready_count`, `preset_cities`.

### `GET /api/projects/presets`

Returns the 11 preset cities (key, name, color, `peak_lst_C`, `month`) used by the Demo tab and Heat & Equity.

### `GET /api/projects/{project_id}`

Portfolio manifest: per-city `status` (`pending` | `processing` | `ready` | `error`), `lst_stats`, `summary`, and `vector_layer` URLs when ready.

### `POST /api/projects/{project_id}/cities`

Register a city by address (geocoded). Use a US **city and state** string (e.g. `"Round Rock, TX"`). The server tries the Census Geocoder first, then OpenStreetMap Nominatim, then hardcoded centroids for the 11 demo cities.

```json
{ "address": "Round Rock, TX" }
```

**Errors:** `400` when the address cannot be resolved (typo, missing state, or non-US place).

### `POST /api/projects/{project_id}/cities/{city_key}/lst`

Upload Landsat GeoTIFFs (`ST_B10`, `SR_B4`, `SR_B5`) for one registered city. Runs LST pipeline, loads census tracts, zonal-joins `lst_mean_C` per tract.

**Content-Type:** `multipart/form-data` â€” field `files` (one or more `.tif`).

**Response:** updated project manifest.

### `GET /api/projects/{project_id}/cities/{city_key}/geojson`

Tract GeoJSON with ACS fields plus `lst_mean_C`, `lst_max_C` (ready cities only).

### `GET /api/projects/{project_id}/cities/{city_key}/gpkg`

Download enriched `.gpkg` for the city.

## City layers (Heat & Equity)

### `POST /api/city-layers`

Load census tract boundaries, ACS demographics, and map layer metadata for a US city/county. Uses the same geocoding chain as project city registration (Census â†’ Nominatim â†’ demo centroids). Prefer `City, ST` format.

**Content-Type:** `application/json`

```json
{ "address": "Round Rock, TX" }
```

**Requires:** `CENSUS_API_KEY` in environment.

**Response:** `CityLayersResponse` with `summary`, `vector_layer` (GeoJSON/GPKG URLs), `worldpop`, and `map_layers`.

### `GET /api/city-layers/demo-portfolio?warm=true`

Cached city-layers payloads for all 11 demo cities. Pass `warm=true` to prefetch missing entries in the background.

### `GET /api/city-layers/vector/{token}/geojson`

Returns tract polygons with ACS attributes for MapLibre (from cached GeoPackage).

### `GET /api/city-layers/vector/{token}/download`

Downloads the cached `.gpkg` file (SQLite/GeoPackage).

### `GET /api/city-layers/map/{token}/preview`

Returns PNG choropleth when `CITY_LAYERS_RENDER_PNG=true` in `.env` (optional; MapLibre is default).

### `GET /api/city-layers/worldpop/{token}/preview`

Returns PNG preview of WorldPop gridded population clipped to county bounds.

Tract queries run automatically inside `POST /api/followup` when `context.tract_layer_token` is set.

## Error responses

FastAPI returns JSON:

```json
{ "detail": "Human-readable error message" }
```

Common cases:

| Status | Cause |
|--------|-------|
| 400 | Missing files, ungeocodable address, invalid token |
| 422 | Invalid JSON body (e.g. `context.raster` sent as `null` instead of a string) |
| 429 | Chat rate limit exceeded |
| 502 | External API unreachable (Census Geocoder, Nominatim, or Census ACS) |
| 500 | Pipeline or unexpected server error |
