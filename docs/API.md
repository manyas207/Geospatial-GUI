## Analysis models

### `GET /api/models`

Lists registered analysis models (inputs, dashboard type, enrichment behavior).

**Response:**

```json
{
  "models": [
    {
      "id": "lst",
      "label": "Land Surface Temperature",
      "description": "Compute land surface temperature from Landsat...",
      "dashboard": "equity",
      "vector_join": "tract_zonal",
      "primary_metric": "mean_C",
      "input_schema": [
        {
          "name": "landsat_bands",
          "label": "Landsat GeoTIFFs",
          "required": true,
          "accept": ".tif,.tiff,.geotiff,.gtiff",
          "hint": "ST_B10 (thermal), SR_B4 (red), and SR_B5 (NIR) for the same scene"
        }
      ]
    }
  ]
}
```

## Follow-up chat

### `POST /api/followup`

Ask Ollama about the current dashboard or city context.

**Content-Type:** `application/json`

```json
{
  "question": "Which city is hottest?",
  "context": {
    "model": "equity",
    "analysis_model": "lst",
    "summary": "11-city urban heat and equity demo.",
    "stats": { "active_city": "Phoenix, AZ" },
    "logs": "",
    "raster": "",
    "tract_layer_token": "Z3Bn...",
    "project_id": null,
    "demo_cities": [{ "name": "Phoenix, AZ", "peak_lst_C": 42.3 }],
    "demo_overview": { "hottest_city": "Phoenix, AZ", "peak_lst_C": 42.3 }
  }
}
```

| Field | Meaning |
|-------|---------|
| `model` | Dashboard shell type (`equity` today) |
| `analysis_model` | Registry model id that produced project stats (e.g. `lst`); optional in demo mode |

**Response:**

```json
{ "answer": "Phoenix, AZ has the highest demo LST (42.3°C)." }
```

**Rate limits:** Per client IP (default 15 requests per 60 seconds). Configure with `CHAT_RATE_LIMIT_MAX`, `CHAT_RATE_LIMIT_WINDOW`, and `CHAT_MAX_QUESTION_LENGTH` in `.env`. Returns `429` with `Retry-After` when exceeded.

**Project context:** When `context.project_id` is set, the server attaches `city_comparison` from the project manifest. For tract queries, use `tract_layer_token` as either a city-layers cache token or `{project_id}:{city_key}` for enriched project GPKGs.

**Demo context:** When `context.demo_cities` is set (no `project_id`), cross-city comparison uses placeholder LST values.

**Context fields:** `raster` must be a string (use `""` when no GeoTIFF path is available). Optional fields may be omitted or set to `null` except where noted in `DashboardContext`.

**Errors:** `400` (empty or overlong question), `422` (invalid request body — e.g. wrong type for `context.raster`), `429` (rate limit), `500` (server error).

## Multi-city analysis projects

### `POST /api/projects`

Create an empty multi-city project.

```json
{ "name": "My LST City Project", "model_id": "lst" }
```

`model_id` is optional (default `lst`). Must match a model from `GET /api/models`.

**Response:** manifest with `id`, `model_id`, `cities`, `ready_count`, `preset_cities`.

The Ask tab auto-creates a project on first use if none is stored locally. There is no list-all-projects endpoint — one active project id is kept in browser `localStorage`.

### `GET /api/projects/presets`

Returns the 11 preset cities (key, name, color, `peak_lst_C`, `month`) used by the Demo tab and Heat & Equity.

### `GET /api/projects/{project_id}`

Portfolio manifest: per-city `status` (`pending` | `processing` | `ready` | `error`), `model_id`, `run_stats` (and `lst_stats` alias for LST), `summary`, and `vector_layer` URLs when ready.

### `POST /api/projects/{project_id}/cities`

Register a city by address (geocoded). Use a US **city and state** string (e.g. `"Round Rock, TX"`). The server tries the Census Geocoder first, then OpenStreetMap Nominatim, then hardcoded centroids for the 11 demo cities.

```json
{ "address": "Round Rock, TX" }
```

**Errors:** `400` when the address cannot be resolved (typo, missing state, or non-US place).

### `POST /api/projects/{project_id}/cities/{city_key}/run`

Run a registered analysis model on uploaded files for one city.

**Query:** `model` — model id (default `lst`). Must match `GET /api/models`.

**Content-Type:** `multipart/form-data` — field `files` (one or more files per model `input_schema`).

**Response:** updated project manifest (city `status` becomes `processing` immediately).

The run executes in a **background task**. Poll `GET /api/projects/{project_id}` until the city `status` is `ready` or `error`. The Ask tab does this automatically and shows model-specific progress messages (LST vs OBIA step labels).

For LST (`model=lst`), upload Landsat GeoTIFFs (`ST_B10`, `SR_B4`, `SR_B5`). The pipeline zonal-joins `lst_mean_C` per tract when enrichment applies.

### `GET /api/projects/{project_id}/cities/{city_key}/geojson`

Tract GeoJSON with ACS fields plus `lst_mean_C`, `lst_max_C` (ready cities only).

### `GET /api/projects/{project_id}/cities/{city_key}/gpkg`

Download enriched `.gpkg` for the city.

## City layers (Heat & Equity)

### `POST /api/city-layers`

Load census tract boundaries, ACS demographics, and map layer metadata for a US city/county. Uses the same geocoding chain as project city registration (Census → Nominatim → demo centroids). Prefer `City, ST` format.

**Content-Type:** `application/json`

```json
{ "address": "Round Rock, TX" }
```

**Requires:** `CENSUS_API_KEY` in environment.

**Response:** `CityLayersResponse` with `summary`, `vector_layer` (GeoJSON/GPKG URLs), and `map_layers`.

### `GET /api/city-layers/demo-portfolio?warm=true`

Cached city-layers payloads for all 11 demo cities. Pass `warm=true` to prefetch missing entries in the background.

### `GET /api/city-layers/vector/{token}/geojson`

Returns tract polygons with ACS attributes for MapLibre (from cached GeoPackage).

### `GET /api/city-layers/vector/{token}/download`

Downloads the cached `.gpkg` file (SQLite/GeoPackage).

### `GET /api/city-layers/map/{token}/preview`

Returns PNG choropleth when `CITY_LAYERS_RENDER_PNG=true` in `.env` (optional; MapLibre is default).

Tract queries run automatically inside `POST /api/followup` when `context.tract_layer_token` is set.

## Error responses

FastAPI returns JSON:

```json
{ "detail": "Human-readable error message" }
```

Common cases:

| Status | Cause |
|--------|-------|
| 400 | Missing files, ungeocodable address, invalid token, unknown `model` id |
| 422 | Invalid JSON body (e.g. `context.raster` sent as `null` instead of a string) |
| 429 | Chat rate limit exceeded |
| 502 | External API unreachable (Census Geocoder, Nominatim, or Census ACS) |
| 500 | Pipeline or unexpected server error |

## Environment variables

See `.env.example` and [DATA.md](DATA.md). Key variables:

| Variable | Affects |
|----------|---------|
| `OLLAMA_*` | Follow-up chat |
| `CENSUS_API_KEY` | `/api/city-layers` demographics |
| `CITY_LAYERS_RENDER_PNG` | Optional server-side map PNGs |
| `OBIA_N_SEGMENTS` | OBIA SLIC segment count (default `50000`) |
| `OBIA_MIN_CLASS_FRACTION` | OBIA labeled-segment purity (default `0`) |
